## 阶段 1：转换区

转换区是用户的入口，负责收集输入并调用 subconverter 生成完整配置。

### 1.1 落地节点信息输入区（左侧）

- 位置：左侧
- 用途：输入落地节点/订阅信息
- 输入方式：
  - 直接粘贴订阅链接（URL）
  - 直接粘贴节点链接（`ss://`、`vmess://`、`trojan://`、`socks5://` 等）
  - 手动添加 SOCKS5 节点信息（**提供专用表单**）
- 显示规则：
  - 每条信息占一行
  - 左侧标记行号
  - 内容不换行，必要时输入窗口允许横向滚动
- 📌 **允许混合输入**：用户可在同一输入框内混合粘贴订阅 URL 与节点链接

### 1.2 中转信息输入区（右侧）

- 位置：右侧
- 用途：输入中转节点/订阅信息
- 输入方式：
  - 直接粘贴订阅链接（URL）
  - 直接粘贴节点链接
  - 手动添加端口转发服务信息（**提供专用表单**）
- 显示规则：
  - 每条信息占一行
  - 左侧标记行号
  - 必要时输入窗口允许横向滚动
- 📌 **允许混合输入**

### 1.3 高级选项（右下角）

- 触发：点击"高级选项"按钮展开
- 包含：
  - **客户端**：下拉菜单（当前仅 Clash/Mihomo）
  - **转换模板**：下拉菜单
  - **Emoji**：checkbox
  - **启用 UDP**：checkbox
  - **跳过证书验证**：checkbox

### 1.4 自动识别按钮

- 样式：大号横向按钮，视觉突出
- 功能：
  1. 调用后端，将用户输入传递给 subconverter 生成完整配置
  2. 解析生成的配置，自动识别/提取落地节点列表
  3. 自动识别/填写配置区的前置/中转策略组信息
  4. 引导用户平滑过渡到**阶段 2：配置区**

## 阶段 2：配置区

配置区以列表形式展示和编辑每条落地节点的链式代理/端口转发配置。

### 2.1 整体布局

- 类似表格/列表，每一 row 为一条配置
- 列结构如下：

| 列 | 内容 | 交互 |
|----|------|------|
| 第一列：落地节点 | 从转换区获取的落地节点逐行排列 | 可手动添加；提供节点列表选择 |
| 第二列：中转方式 | `链式代理` 或 `端口转发` | 可点击切换 |
| 第三列：前置/中转节点 | 自动匹配区域策略组 | 自动填写；失败时手动选择策略组/节点 |

### 2.2 落地节点列（第一列）

- 数据来源：阶段 1 自动识别后提取的落地节点
- 可手动添加新的落地节点
- 提供节点列表供用户选择（从完整配置中的 proxies 拉取）

### 2.3 中转方式列（第二列）

- 默认值：链式代理
- 可点击在 `链式代理` ↔ `端口转发` 之间切换
- 切换后第三列的可选项应对应变化

### 2.4 前置/中转节点列（第三列）

- **自动匹配逻辑**：从落地节点名称中识别区域信息（如"香港""日本""美国"等），自动添加对应区域的策略组
- **自动判断失败时**：由用户手动选择策略组或节点
- 📌 可选来源：
  - 策略组（proxy-groups 中的条目）
  - 单个节点（proxies 中的条目）

### 2.5 每行高级选项

- 触发：展开按钮，向左滑动展开高级选项列
- 包含：
  1. **替换 / 创建新节点**：选择是替换原节点还是创建一条新的节点
  2. **是否改名**：允许自定义节点名
  3. **复制本行**：复制当前行的配置到新行
  4. **+/- 行**：增加或删除配置行

### 2.6 生成按钮

- 样式：大号横向按钮，视觉突出
- 功能：将配置区的所有配置信息提交后端，生成最终 YAML
- 引导用户进入**阶段 3：生成区**

## 阶段 3：生成区

### 3.1 链接展示区（第一行）

- 展示生成的链接
- 可点击区域复制链接
- 右侧提供"使用短链接"选项（toggle/checkbox）

### 3.2 操作按钮（第二行）

- 三个大号按钮依次排列：
  1. **打开**：在新标签页打开 YAML
  2. **复制**：复制 YAML 内容到剪贴板
  3. **下载**：下载为 `.yaml` 文件

### 3.3 日志区（最下方）

- 可折叠/展开
- 展示处理过程的日志信息（解析、生成、修改步骤的输出与警告）

---

## 跨阶段交互与导航

- 三个阶段**依次展开**（或使用 stepper/tab 导航）
- 用户可以回退到前一阶段修改输入并重新执行
- 阶段间的数据传递通过前端状态管理（内存中）

---

## 响应式设计

- 桌面端：左右双列布局（阶段 1 的两个输入区并排）
- 移动端：单列堆叠布局
