# 05 - Review 结论

> 本文档汇总规格中的遗漏、逻辑问题与待澄清项，供实现与迭代参考。

---

## 一、遗漏

| 项目 | 说明 |
|------|------|
| 代理链接解析 | `ss://`、`vmess://` 等 URI 的解析与转换为 Mihomo 节点格式，需实现或对接现有解析库 |
| 统一生成链路落地 | 文档已统一为“模板 + subconverter”单一路径，但实现侧仍需真正删除旧的“检测通用配置后分流”逻辑。 |
| 填表模式 UI | 04 中 `source.type="socks5_form"` 定义的字段需有对应前端表单与校验 |
| 文件上传 API | `/api/parse_config` 或等效接口未实现。当前应只承担节点提取与前端引导，不再承担“通用配置检测”。 |
| 角色确认 UI | 前端需要明确支持：选择哪些节点是中转、哪些节点是落地、以及“落地已包含在中转中”的勾选与校验。 |
| 元信息持久化 | 需明确完整配置之外如何保存 `selected_transit_names[]`、`selected_landing_names[]`、`landing_included_in_transit`、`forward_relays[]` 等信息，供下一阶段继续编辑。 |
| provider 为空 | 无 `proxy-providers` 时，选项二应隐藏或禁用。02 澄清：不需要解析 proxy-providers 内节点，仅需 use 引用 |
| 协议字段映射 | 端口转发时，所有协议均只需替换 server 与 port 字段，reality 走“复制新节点”，见 03 |
| 旧 query 入参弃用 | 旧版将业务参数塞进 query 的方式一律弃用；重构后统一 `POST` + JSON body（见 04）。 |
| 默认模板配置项 | 当前规格默认依赖内置模板，但模板内容、版本管理、回滚方式仍需在实现前落定。 |
| 修改方式字段 | API 必须显式要求 `modification_mode`，避免后端推断用户意图（链式/端口转发/同时启用）（见 03 与 04）。 |
| 角色确认字段 | API 必须显式要求 `selected_transit_names[]` 与 `selected_landing_names[]`（自动识别只能作为建议默认值），避免误改写普通节点（见 02、03 与 04）。 |
| 链式 dialer 成员选择字段 | 链式代理需要由前端把“每个落地节点 dialer 组成员”显式传入（如 `chain_dialer_membership`），后端不应猜测 UI 选择（见 03 与 04）。 |

## 二、逻辑 / 一致性问题

| 项目 | 建议 |
|------|------|
| 7 区域分类 | 使用「先匹配 6 类，未命中则为 Other」，避免超长负向断言 |
| 节点来源 | 明确选项一仅针对 `proxies`；若支持 provider 节点，需单独设计 |
| 组引用 | 新建 dialer 的 `proxies` 是否可包含已有 proxy-group 名称，需在 03 中明确 |
| 单源双角色场景 | 当 `landing_included_in_transit=true` 时，需明确前端和后端如何防止同一节点被错误地同时当作中转与落地使用。 |

## 三、待澄清

| 项目 | 说明 |
|------|------|
| 手动添加落地 | 具体交互：多选列表、搜索、确认按钮等 |
| 中转与落地是否允许重叠 | 业务上是否允许同一节点同时出现在 `selected_transit_names[]` 与 `selected_landing_names[]`，若允许，哪些修改模式下有效，需定稿。 |
| 端口转发 relay 多选策略 | `forward_relays[]` 多个时，是“只允许选一个生成单份配置”，还是“每个 relay 生成一份输出/订阅多条”，需要产品层面定稿（03 已说明两种策略）。 |
| 自定义模板上线节奏 | 当前规格保留扩展点，但本迭代是否完全不做自定义模板 UI/API，需要确认。 |

## 四、与现有实现的差异

| 现有能力 | 重构后预期 |
|----------|------------|
| 仅 URL 拉取 | 新增文件上传、粘贴、端口转发（仅地址+端口） |
| 单一输入 | 升级为“中转输入 + 落地输入”的双角色模式；若勾选“落地已包含在中转中”，则允许单源内显式划分角色 |
| 落地仅来自订阅 | 新增代理链接、填表、手动指定 |
| 简单关键字识别 | 扩展 7 区域正则 + 手动指定落地 |
| 通用配置检测分流 | 删除旧的“检测中转是否包含通用配置 → 选择生成路径”思路，统一改为模板 + subconverter 生成完整配置 |
| 仅 dialer-proxy | 支持 dialer-proxy + 端口转发（含协议字段映射） |
