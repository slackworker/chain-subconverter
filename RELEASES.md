# Release Notes

## 3.0 Alpha

> 当前对外内测线，更新时间：2026-05-01

### 发布信息

- 当前对外版本阶段：`3.0 Alpha`
- 当前发布线：`release/3.0`
- 当前默认 UI 基线：`scheme default`
- 默认应用镜像：`ghcr.io/slackworker/chain-subconverter:alpha-latest`
- 推荐发布记录：`v3.0.0-alpha.N` 等不可变 tag
- 默认访问入口：`/`

`3.0 Alpha` 是当前对外内测发布线，用于收口第三方设备部署、前端主流程和恢复链路；该阶段仍不是 Beta 或正式版。`alpha-latest` 可继续作为便捷镜像入口，但回归记录与反馈应逐步绑定到不可变版本 tag。

### 主要变更（相对 2.0）

本版本主要变更如下：

- **前后端单镜像集成（app）**：2.0 更多依赖手工拼接运行方式、部署路径分散；3.0 Alpha 将前端与后端统一收口到同一个 `app` 镜像，单服务入口即可提供 UI 与 API。
- **Docker Compose 联合部署（app + subconverter）**：3.0 Alpha 提供统一 Compose 路径编排 `app` 与 `subconverter`，第三方设备可直接冷启动。
- **可视化下拉菜单配置**：2.0 用户更容易落入“手填参数 + 反复试错”的流程；3.0 Alpha 核心配置转为可视化选择，交互更直观，配置门槛更低。
- **自动读取模板区域策略组并填充落地节点**：3.0 Alpha 会自动读取模板中的区域策略组，满足命名条件的落地节点会自动填充到配置中，减少手工维护成本与命名出错概率。
- **支持短链分享与复用**：2.0 主要依赖长链接分发，复制与传播成本较高；3.0 Alpha 可生成短链接，便于跨设备分享、记录和复用。
- **支持反向解析（恢复配置）**：2.0 通过已有链接回到可编辑状态的能力有限；3.0 Alpha 支持恢复当前配置，方便继续编辑、回看和复用。

### 已知限制与边界

- 默认 `/` 已固定为 Alpha 当前对外入口；`/ui/a`、`/ui/b`、`/ui/c` 继续保留用于实验与对照，不作为 Alpha 发布阻塞项。
- 当前版本未针对移动端进行专门适配，移动端体验不作为 Alpha 阶段验收标准。
- Alpha 反馈仍在持续收口，尚未进入 Beta 候选冻结。
- 对外发布、第三方设备最小回归与反馈记录模板见 [docs/testing/alpha-release.md](docs/testing/alpha-release.md)。
- 2.0 及更早历史发布说明见 [_legacy/RELEASES.md](_legacy/RELEASES.md)。
