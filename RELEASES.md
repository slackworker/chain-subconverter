# Release Notes

## 当前发布口径

> 当前滚动发布说明，更新时间：2026-05-17

### 分支与标签

- `dev`：开发线；仅手动发布 `dev-latest`
- `beta`：预发布线；进入 Beta 阶段后默认发布 `beta-latest`，并可附加 `vX.Y.Z-beta.N`
- `main`：稳定线；当前默认发布 `latest`，正式版时可附加 `vX.Y.Z`
- 当前默认 UI 基线：`scheme default`
- 当前默认应用镜像：`ghcr.io/slackworker/chain-subconverter:latest`
- 默认访问入口：`/`

当前 3.0 仍处于开发后期与发布整理阶段。仓库的发布模型已收口为 `dev / beta / main` 三线；当前对外默认使用 `main` 产生的滚动标签 `latest`。`beta-latest` 与版本号标签保留给后续 Beta / 正式版发布节点，不在当前阶段强制启用。

### 主要变更（相对 2.0）

本版本主要变更如下：

- **前后端单镜像集成（app）**：2.0 更多依赖手工拼接运行方式、部署路径分散；3.0 当前基线已将前端与后端统一收口到同一个 `app` 镜像，单服务入口即可提供 UI 与 API。
- **Docker Compose 联合部署（app + subconverter）**：3.0 当前基线提供统一 Compose 路径编排 `app` 与 `subconverter`，第三方设备可直接冷启动。
- **可视化下拉菜单配置**：2.0 用户更容易落入“手填参数 + 反复试错”的流程；3.0 当前基线的核心配置已转为可视化选择，交互更直观，配置门槛更低。
- **自动读取模板区域策略组并填充落地节点**：3.0 当前基线会自动读取模板中的区域策略组，满足命名条件的落地节点会自动填充到配置中，减少手工维护成本与命名出错概率。
- **支持短链分享与复用**：2.0 主要依赖长链接分发，复制与传播成本较高；3.0 当前基线可生成短链接，便于跨设备分享、记录和复用。
- **支持反向解析（恢复配置）**：2.0 通过已有链接回到可编辑状态的能力有限；3.0 当前基线支持恢复当前配置，方便继续编辑、回看和复用。

### 已知限制与边界

- 默认 `/` 已固定为当前对外入口；`/ui/a`、`/ui/b`、`/ui/c` 继续保留用于实验与对照，不作为当前发布整理阻塞项。
- 当前版本未针对移动端进行专门适配，移动端体验不作为当前阶段验收标准。
- `beta` 分支与版本号标签规则已预留，但尚未进入正式 Beta 冻结与版本化发布。
- 对外发布、第三方设备最小回归与反馈记录模板见 [docs/testing/release-runbook.md](docs/testing/release-runbook.md)。
- 2.0 及更早历史发布说明见 [_legacy/RELEASES.md](_legacy/RELEASES.md)。
