# chain-subconverter 3.0 发布说明

> 本文档是 **3.0** 的对外发布说明：相对 2.0 你能用到什么、当前发布到哪一步。更早版本见 [_legacy/RELEASES.md](_legacy/RELEASES.md)。  
> 更新时间：2026-05-18

## 版本状态

3.0 仍在发布整理阶段，功能已可自用部署。当前推荐镜像：`ghcr.io/slackworker/chain-subconverter:latest`（`main` 分支持续更新）。正式 Beta / 版本号标签规则已预留，尚未冻结发布。

部署与访问方式见 [deploy/README.md](deploy/README.md)：装好 Compose 后打开 `http://<设备地址>:<端口>/` 即可使用。

## 相对 2.0，新在哪里

- **更好部署**：前后端打成一个服务镜像；Compose 一条路径拉起应用与转换组件，在 NAS、软路由等设备上冷启动即可用。
- **更好配置**：核心选项改为网页下拉与表单，少用手填参数、少试错。
- **更省手工**：按模板自动识别区域策略组，符合条件的落地节点会自动填入，减少对照模板手抄名称。
- **更好分享**：生成结果可变成短链或短 ID，复制发给另一台设备即可打开同一套配置。
- **更好续编**：把长链、短链或 short ID 贴回页面即可恢复编辑；落地、中转与订阅信息跟着链接走，一般不必另存一份节点地址列表。

## 使用上请注意

- 默认入口为 `/`；`/ui/a`、`/ui/b`、`/ui/c` 仅作实验对照，日常请用 `/`。
- 未针对手机浏览器专门优化，小屏体验不保证。
- 公开预览或他人托管实例请自行评估可用性与隐私；生产环境建议自部署。
